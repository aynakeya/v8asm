## Fixed This Iteration

- `tests/decomp_rounds/run_version_matrix.sh` now enumerates executable
  `tests/decomp_rounds/bin_cache/*/v8asm` binaries automatically. This prevents
  the default matrix from silently skipping cached builds such as 10.8, 11.4,
  13.2, and the 13.2 no-static-roots probe.
- The default matrix no longer hardcodes a local V8 `out/` binary. New local
  probes should be passed explicitly with `VERSION_MATRIX_V8ASM_BINS` until
  they are copied into the bin cache.
- The matrix wrapper no longer passes `--snapshot_blob` to metadata commands
  (`version` and `build-args`). The previous wrapper polluted stdout/stderr
  for old cached binaries by forcing startup snapshot initialization while
  collecting metadata.

## Mistake Analysis

I previously focused on the v8asm binary fix for `version`/`build-args`, but
left the matrix wrapper free to prepend `--snapshot_blob` to metadata commands.
That meant old cached binaries could still initialize a startup snapshot and
emit warnings while the matrix was only trying to read metadata. The corrected
rule is stricter: metadata commands must never receive a snapshot override,
independent of whether the binary itself has been rebuilt with the early-return
fix.

I also left the default matrix hardcoded to a subset of cache directories. That
created a false sense of coverage after adding more cached builds. The default
now enumerates the cache directory so adding a cached v8asm automatically adds
it to self-cache and Node/bytenode compatibility checks.

## Remaining Gaps

- This iteration did not rebuild every patch in both Electron and Node modes.
  It validated the current cached binaries and the patch text invariants.
- Cached binaries older than the 13.2/13.6 rebuilds do not print
  `v8_enable_static_roots`; the patch files now include that output, but those
  old caches need rebuilds before their metadata can expose it.
- Only Electron 34.3.0 is currently available under
  `/home/aynakeya/workspace/tmp/v8test/electron-cache/`, so direct
  Electron-generated bytenode validation currently covers the 13.2 Electron
  line. Other Electron-flavored cached binaries still need their matching
  Electron packages/snapshots for full external Electron `.jsc` validation.

## 2026-06-28 13.4 Static-Roots Atom Probe

The 13.4 Node-style build compiled earlier in the day reports:

```text
version: 13.4.114.21
v8_enable_pointer_compression=false
v8_enable_static_roots=false
```

It passes self `asm`/`checkversion`/`disasm`/level-4 decompile with its own
generated `snapshot_blob.bin`, but it cannot initialize the Atom context
snapshot from `v8context/v8_context_snapshot.bin`:

```text
Check failed: false == fixed_offset.
```

That failure comes from V8's `read-only-deserializer.cc` check:

```text
CHECK_EQ(V8_STATIC_ROOTS_BOOL, fixed_offset)
```

So this is a build-flag/snapshot-shape mismatch, not a cached-data header
problem. I built a separate 13.4 Electron-style static-roots probe on the
`13.4.114.21` checkout, with the official environment and fixed `autoninja -j10`:

